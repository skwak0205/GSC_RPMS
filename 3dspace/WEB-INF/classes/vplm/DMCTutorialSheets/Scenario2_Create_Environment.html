Create a new environment from Test_and_Try

0. Introduction

This help sheet describes how to create a new environment from a copy of the Dassault Systemes VPM environment, and how to modify it for referencing a new customization inside in order to be able to test the customization from CATIA.

At this starting point, you are supposed to have created a new customization, or open an existing one.

Click to restart
 
 
1. Create a new Environment from an existing one

Select the Environments tab page on the left side, and select the Environment root node in the tree.

Create a new environment using the "Insert environment..." command from the contextual menu.

In the "Create a new environment" panel, enter the name for the new environment (ex: MyVPM)

Click on the button "Import from another environment".

In the "Import an environment" panel, select an existing environment from the list "select and environment to import" (ex: select VPM), then click on OK.

You are now back on the "Create a new environment" panel. Click on Create to create the new environment.

Click to complete
 
 
2. Remove existing customization for same modelers

If "Authoring modelers" list already contains existing customization for the same modelers as your customization, you need to remove it. This is mandatory before adding your own customization for the same modeler.

Select the existing customization from the "Authoring modelers" list and click on <- button.

Select the existing customization from the "Usable modelers" list and click on <- button.

Click on Apply to update the environment.

Click to complete
 
 
3. Add your customization to the new environment

Select your customization name from the first list, and click on -> button to put it to the "Authoring modelers" list.

Click on Apply to update the environment.

Then the new customization added in as authoring modeler is automatically also added to the usable modelers list.

Click to complete
 
 
Save environments

Select the "File/Save/Environment..."

Environments are automatically saved in the VPMAdminV6GA.adm file under the Working folder specified in the Tools/Options (General tab page) parameters.

Click to complete
 
 
You have successfully finished this tutorial.
