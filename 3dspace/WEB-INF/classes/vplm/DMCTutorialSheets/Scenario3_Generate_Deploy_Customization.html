Generate files for customization and Deploy

0. Introduction

This help sheet describes how to generate all files requested for a new customization, and how to prepare the customization deployment to the Enovia VPM Server.

At this starting point, you are supposed to have created a new customization and declared it in an environment.

Click to restart
 
 
1. Generate all files associated to the customization

Select "Tools/Generate..." command. A list displays all the customization located in your Working folder (the Working folder is specified in the Tools/Options (General tab page) parameters).

Select one or more customization from the list.

Click to complete
 
 
2. Files for updating server database with the new customization data model

Select Generate MQL files checkbox.

Click to complete
 
 
3. Files for P&O access to the customization

Select Generate Default Mask checkbox: A mask file with default access right to the customization types and attributes is generated.

Select Compilation checkbox: a compilation files with all the masks, including the default one for the new customization is generated.

Click on Generate. Messages and errors are displayed in the traces zone.

Click to complete
 
 
4. Backup before publication on server

This step is optional but recommended: it will copy in your DMC Working Folder files that are going to be modified during the next publication step.

Select "Tools/Backup..." command.

Click on Backup.

Click to complete
 
 
5. Publish a customization

Select "Tools/Publish..." command. A list displays all the customization located in your Working folder (the Working folder is specified in the Tools/Options (General tab page) parameters).

Select one or more customization from the list.

Click on Publish. Messages and errors are displayed in the traces zone.

Click to complete
 
 
6. Launch Warutil

 

Select "Tools/Warutil..."

Click on Launch.

On Windows platform, warutil interactive application is launched. Follow steps of the tool.

On Unix platform, warutil batch tool is launched in DMC panel. Enter response of all the warutil questions in the dedicated text field and validate with Enter button. Messages and errors are displayed in the traces zone.

At the end of this step, new .war and .ear files are generated the distrib directory.

Click to complete
 
 
7. Deploy on Server

 

Select "Tools/Deploy on Server..."

This is only an information step in DMC tool, and deployment has to be done manually with 2 steps:

1. use the appropriate Application Server administration console to deploy the .war file created in the previous step.

2.use the MQL console from MatrixOne to execute all mql files located in ENOVIA Collaboration Platform Server (matrixRmi)/DMCmql directory

Click on OK when done.

Click to complete
 
 
8. Commit deployment

 

Select "Tools/Commit..."

This step is mandatory in DMC tool after manual deployment of .war file and mql scripts execution.

Click on Commit.

Click to complete
 
 
You have successfully finished this tutorial.
